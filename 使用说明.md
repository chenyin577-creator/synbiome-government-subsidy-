# 研发补贴平衡台账：极简操作说明

这套工具只看一件事：

**杭州微新和深圳微智的研发投入够不够，能不能顺利拿到补贴。**

可以把它理解成“给水桶加水”：

- 每个补贴项目是一个水桶。
- 政府要求研发投入，就是桶上的最低水位线。
- 每月新增研发投入，就是往桶里加水。
- 哪个桶水位不够，就优先往哪个桶加。

## 第一步：先把水桶放好

进入左侧 **项目设置**。

什么时候做：  
新项目开始申报，或者以前已经申报过但系统里还没有。

怎么做：

1. 点 **新增项目**。
2. 选择主体：杭州微新或深圳微智。
3. 填项目名称和项目编号。
4. 填 **政府要求研发投入**。
5. 填 **补贴申请金额** 和 **补贴已到账金额**。
6. 保存。

如果项目很多，不想一个个填，就点 **导入历史项目**，把表格内容粘进去。

这一页只在前期设置项目时用，平时不用天天进。

## 第二步：每月数据进来

进入左侧 **每月录入**。

什么时候做：  
每月会计做完账以后。

当前临时做法：

1. 选本月月份。
2. 找到对应项目。
3. 在 **本月新增研发投入** 里填金额。
4. 点 **保存本月数字**。

后续正式做法：

系统从会计系统、邮箱或接口自动取得明细账，不再让会计每月重复搬数据。

注意：

- 如果这笔研发投入应该算杭州，就填到杭州微新的项目里。
- 如果应该算深圳，就填到深圳微智的项目里。
- 如果暂时不确定，先问项目负责人，不要随便放。

## 第三步：看哪个桶还没满

进入左侧 **平衡总览**。

重点只看两个数字：

1. **研发投入达标率**  
   这个越接近 100% 越好。  
   低于 100%，说明这个项目还没达到政府要求。

2. **研发投入缺口**  
   这个就是还差多少钱。  
   缺口越大，越要优先安排后续研发支出。

判断方法很简单：

- 杭州缺口大，就优先考虑把合规研发费用放到杭州微新。
- 深圳缺口大，就优先考虑把合规研发费用放到深圳微智。
- 如果两个都缺，就先看哪个项目更快到申报或验收节点。

## 谁负责什么

项目负责人负责：

- 新增项目
- 导入历史项目
- 确认研发费用应该归到哪个项目

会计负责：

- 提供真实会计样表
- 确认会计系统是否支持 API、定时邮件或自动导出
- 在自动化没上线前，临时核对新增研发投入金额

管理层负责：

- 接收系统推送的简版月报
- 决定后续研发支出优先放杭州还是深圳

## 管理层只看什么

CEO/CFO 不需要看明细表。

每月给管理层看的页面只保留三类信息：

- 是否预警：红色、橙色、绿色。
- 大概差多少：研发投入缺口、补贴到账率。
- 本月要不要处理：系统列出最重要的三件事。

明细账、费用口径、项目归属、人员费用分摊，都由后台人员处理。

正式自动化完成后，管理层月报由系统自动生成并发送。

截图或另存为 HTML 只能作为当前临时演示方式，不作为长期每月动作。

## 下一版：系统自动取得账表

下一版接通邮箱或接口后，会计不用重复填表。

目标动作：

1. 会计系统通过 API、定时邮件或自动导出提供明细账。
2. 系统自动读取附件或接口数据。
3. 系统自动识别研发费用。
4. 申报负责人只审核系统不确定的记录。
5. CEO/CFO 只收到管理层简报。

如果暂时没有接口，可以先用测试邮件跑通，但不能长期依赖人工下载、上传、复制、粘贴。

不要把这个工具当成完整财务系统。

它现在只是一个“防漏补贴”的小台账：

**看杭州和深圳两边有没有达到政府要求，哪里缺就补哪里。**
