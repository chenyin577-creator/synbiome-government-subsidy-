# 邮件闭环资料总索引

## 这份索引怎么用

不要把所有文件一次性发给所有人。

按角色分发，谁负责什么，就看对应文件。

## 管理层需要看

管理层只需要判断是否同意下一版边界。

建议发送：

1. `邮件闭环决策一页纸.md`
2. `管理层月报样张.html`
3. `邮件闭环三方确认回执表.csv`

管理层需要确认：

- 是否使用专用邮箱。
- 是否每月固定发送简报。
- 是否认可 CEO/CFO 只看简版 HTML 月报。
- 是否同意邮件附件导入只是过渡接口，不把人工搬运作为长期流程。

## 会计需要看

会计只需要确认账表能不能导出、字段能不能对应。

建议发送：

1. `会计研发费用明细样表.csv`
2. `附件字段映射确认表.csv`
3. `会计样表填写说明.md`
4. `测试邮件模板.md`
5. `邮件闭环AI原生执行原则.md`

会计需要确认：

- 能否导出 Excel 或 CSV。
- 是否包含日期、主体、科目、摘要、金额等字段。
- 真实账表列名如何对应系统字段。
- 测试邮件能否按模板发送。
- 会计系统是否支持 API、报表订阅、定时邮件或自动导出。

## 张英需要看

张英只需要确认不确定记录怎么审核。

建议发送：

1. `邮件闭环决策一页纸.md`
2. `邮件闭环三方确认回执表.csv`
3. `邮件闭环测试验收流程.md`

张英需要确认：

- 是否由她审核不确定记录。
- 是否重点审核人员费用分摊。
- 是否重点审核项目归属不清的费用。

## IT 或邮箱管理员需要看

IT 或邮箱管理员只负责邮箱和授权密码安全。

建议发送：

1. `邮件闭环AI原生执行原则.md`
2. `邮箱授权安全交接说明.md`
3. `邮件闭环上线前检查清单.md`

IT 或邮箱管理员需要确认：

- 专用邮箱是否创建。
- 专用邮箱是否可以收发邮件。
- 授权密码是否只配置到腾讯云环境变量。
- 授权密码没有发到微信群、网页文件或 GitHub。
- 是否存在会计系统接口、邮箱自动投递或云盘自动投递方案。

## 实施或开发人员需要看

实施人员负责按设计开发和验收。

建议发送：

1. `邮件驱动闭环设计说明.md`
2. `邮件闭环AI原生执行原则.md`
3. `邮件闭环开发启动条件清单.md`
4. `邮件闭环测试验收流程.md`
5. `邮件闭环上线前检查清单.md`
6. `docs/superpowers/specs/2026-06-20-mail-driven-ledger-design.md`
7. `docs/superpowers/plans/2026-06-20-mail-driven-ledger.md`

实施人员需要确认：

- 邮件附件导入是过渡接口，不是长期人工搬运流程。
- 前端只显示结果、预警和异常项，不作为每月固定确认入口。
- 邮箱授权密码不进入前端文件。
- 测试邮件可以端到端跑通。
- HTML 月报可以发送给指定收件人。

## 推荐推进顺序

1. 管理层先确认边界。
2. 会计提供真实样表。
3. 会计完成字段映射确认。
4. IT 或实施人员确认会计系统接口、定时导出、邮箱自动投递能力。
5. IT 创建专用邮箱并准备授权。
6. 用 `邮件闭环开发启动条件清单.md` 判断能否开始开发。
7. 开发人员接入邮箱和附件解析。
8. 用 `邮件闭环测试验收流程.md` 跑通测试。
9. 用 `邮件闭环上线前检查清单.md` 判断是否能试运行。
