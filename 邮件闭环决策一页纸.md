# 邮件闭环决策一页纸

## 要解决的问题

杭州微新和深圳微智两边研发费用必须持续达标，才能稳妥拿到补贴。

现在的问题是：每月靠人工统计，容易晚、容易漏、容易分错主体。

下一版目标：

**会计每月发账表邮件，系统自动导入，负责人只审核不确定项，CEO/CFO 只收简版预警月报。**

## 建议直接确认的默认方案

| 事项 | 默认方案 |
|---|---|
| 专用邮箱 | `subsidy-ledger@synbiome.cn` |
| 附件格式 | Excel 或 CSV |
| 发送时间 | 每月 6 日上午 9 点 |
| 不确定记录审核人 | 张英 |
| 管理层页面 | 只看 HTML 简报，不看明细 |
| 第一版范围 | 只做邮件附件导入，不直连完整会计系统 |

## CEO/CFO 会看到什么

只看五项：

1. 是否预警。
2. 研发投入达标率。
3. 研发投入缺口。
4. 补贴到账率。
5. 本月最重要三件事。

不会看到：

- 单笔费用明细。
- 凭证和发票。
- 附件解析过程。
- 技术日志。

## 会计需要做什么

每月只做三步：

1. 从账套导出研发费用明细账。
2. 保留日期、主体、科目、摘要、金额、凭证号、部门或项目等字段。
3. 发到专用邮箱。

参考文件：

- `会计研发费用明细样表.csv`
- `会计样表填写说明.md`
- `测试邮件模板.md`

## 张英需要做什么

只审核系统判断不准的记录，例如：

- 人员费用分摊。
- 杭州还是深圳归属不明确。
- 项目编号缺失。
- 科目不是研发费用但摘要疑似研发。

## 现在需要负责人拍板

请确认三件事：

1. 是否同意使用专用邮箱 `subsidy-ledger@synbiome.cn`。
2. 是否同意每月 6 日上午 9 点发送管理层简报。
3. 是否同意第一版只支持 Excel 和 CSV，不处理 PDF、图片、凭证包。

## 开发启动条件

满足以下三项后，可以进入正式开发：

1. 专用邮箱已创建，并能生成授权密码。
2. 会计提供一份真实导出的 Excel 或 CSV 样表。
3. 负责人确认本页默认方案。

## 当前不能误解

现在已经完成的是设计确认材料。

现在还没有完成自动收邮件、自动解析附件、自动发送 CEO/CFO 月报。
