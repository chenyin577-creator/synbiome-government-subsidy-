# 邮件闭环上线前检查清单

## 用途

这份清单用于正式开发或上线前逐项确认。

没有全部确认前，不应声称系统已经可以自动收邮件、自动解析附件、自动发送 CEO/CFO 月报。

## 1. 业务规则确认

- [ ] 已按 `邮件闭环资料总索引.md` 分发对应资料。
- [ ] 已确认 `邮件闭环AI原生执行原则.md`。
- [ ] 已按 `邮件闭环开发启动条件清单.md` 确认可以开始开发。
- [ ] 已确认专用邮箱。
- [ ] 已确认月报收件人。
- [ ] 已确认每月发送时间。
- [ ] 已确认第一版只支持 Excel 和 CSV。
- [ ] 已确认不处理 PDF、图片、凭证包。
- [ ] 已确认不做完整会计系统深度直连，但已先查接口或自动导出能力。
- [ ] 已确认张英审核不确定记录。
- [ ] 已确认 CEO/CFO 只看简版 HTML 月报。
- [ ] 已确认前端只展示结果、预警和异常项，不作为每月固定确认入口。
- [ ] 已确认人工导出、上传、复制、粘贴不是长期流程。
- [ ] 已确认会计系统接口、定时导出、邮箱自动投递或云盘自动投递的可行性结论。

## 2. 会计账表确认

- [ ] 会计已提供一份真实导出的 Excel 或 CSV。
- [ ] 真实样表不含不应外发的敏感内容，或已经脱敏。
- [ ] 已用 `附件字段映射确认表.csv` 对照真实列名。
- [ ] 日期、主体、科目、摘要、金额至少可识别。
- [ ] 项目、部门、凭证号、人员姓名、分摊比例等字段已确认是否能导出。

## 3. 邮箱和安全确认

- [ ] 专用邮箱可以正常登录。
- [ ] 专用邮箱可以正常收邮件。
- [ ] 专用邮箱可以正常发邮件。
- [ ] 已生成邮箱授权密码。
- [ ] 授权密码没有发到微信群。
- [ ] 授权密码没有写进网页文件。
- [ ] 授权密码没有上传到 GitHub。
- [ ] 授权密码只配置在腾讯云后台环境变量里。

## 4. 技术环境确认

- [ ] 腾讯云环境变量已配置。
- [ ] GitHub 仓库在放入真实财务数据前已改为 Private。
- [ ] 系统没有设计每月固定的前端确认按钮作为必经步骤。
- [ ] 系统没有设计每月固定的人工上传入口作为主流程。
- [ ] 测试数据和真实数据分清楚。
- [ ] 测试邮件可以被系统读取。
- [ ] 测试附件可以被系统解析。
- [ ] 不确定记录能进入待审核列表。
- [ ] 审核通过后，研发投入金额能更新。
- [ ] HTML 月报能生成。
- [ ] HTML 月报能发送给指定收件人。

## 5. 管理层确认

- [ ] CEO/CFO 已看过 `管理层月报样张.html`。
- [ ] CEO/CFO 认可只看预警等级和关键数字。
- [ ] CEO/CFO 不要求在月报里显示单笔费用明细。
- [ ] CEO/CFO 确认收到月报后能判断是否需要处理。

## 6. 三方确认和测试验收

- [ ] `邮件闭环三方确认回执表.csv` 已完成确认。
- [ ] `邮件闭环测试验收流程.md` 已按步骤跑完。
- [ ] 测试邮件读取成功。
- [ ] 测试附件解析成功。
- [ ] 测试月报发送成功。

## 7. 上线判断

全部勾选后，才可以进入正式上线。

如果还有未勾选事项，只能说：

**邮件闭环仍在测试或准备阶段。**

不能说：

**系统已经自动运行。**
