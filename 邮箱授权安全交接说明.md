# 邮箱授权安全交接说明

## 这份说明解决什么问题

下一版要让系统自动收会计邮件、读取附件、发送管理层月报。

系统需要一个专用邮箱的授权密码。

这个授权密码不是普通登录密码，不能发到微信群，不能写进网页文件，也不能上传到 GitHub。

## 推荐邮箱

`subsidy-ledger@synbiome.cn`

如果暂时还没创建这个邮箱，可以先创建一个同用途的公司邮箱。

不要长期使用个人邮箱。

## 谁来处理

建议由公司邮箱管理员或 IT 处理。

处理人只做两件事：

1. 创建或确认专用邮箱可以正常收发邮件。
2. 生成邮箱授权密码，并配置到腾讯云后台。

## 阿里企业邮箱需要准备的信息

如果公司邮箱由阿里企业邮箱承载，通常需要以下信息：

| 项目 | 建议值 |
|---|---|
| 收件服务器 | `imap.qiye.aliyun.com` |
| 收件端口 | `993` |
| 发件服务器 | `smtp.qiye.aliyun.com` |
| 发件端口 | `465` |
| 邮箱账号 | `subsidy-ledger@synbiome.cn` |
| 邮箱授权密码 | 由邮箱后台生成 |

如果阿里邮箱后台显示的服务器地址不同，以邮箱后台显示为准。

## 腾讯云后台需要配置什么

正式开发时，把下面这些值放到腾讯云后台环境变量里：

| 环境变量名 | 含义 | 示例 |
|---|---|---|
| `LEDGER_IMAP_HOST` | 收件服务器 | `imap.qiye.aliyun.com` |
| `LEDGER_IMAP_PORT` | 收件端口 | `993` |
| `LEDGER_SMTP_HOST` | 发件服务器 | `smtp.qiye.aliyun.com` |
| `LEDGER_SMTP_PORT` | 发件端口 | `465` |
| `LEDGER_MAIL_USER` | 专用邮箱账号 | `subsidy-ledger@synbiome.cn` |
| `LEDGER_MAIL_AUTH_PASSWORD` | 邮箱授权密码 | 不写在文档里 |
| `LEDGER_REPORT_RECIPIENTS` | 月报收件人 | `yin.chen@synbiome.cn,yin.zhang@synbiome.cn,108234704@qq.com` |

## 绝对不要做的事

- 不要把授权密码发给群。
- 不要把授权密码发给不需要操作的人。
- 不要把授权密码写进 `config.js`。
- 不要把授权密码写进任何网页文件。
- 不要把授权密码上传到 GitHub。
- 不要把授权密码放在月报或截图里。

## 安全交接方式

推荐方式：

1. 管理员在腾讯云后台当场输入授权密码。
2. 开发人员只确认环境变量名称是否正确。
3. 开发人员不保存、不记录、不转发授权密码。

如果必须远程交接，建议用企业内部认可的密码工具，不建议用微信、邮件正文或普通文档传密码。

## 配好以后怎么验证

正式开发阶段，用一封测试邮件验证：

1. 会计发送测试邮件到专用邮箱。
2. 邮件带 Excel 或 CSV 附件。
3. 系统能读取这封邮件。
4. 系统能识别附件字段。
5. 系统生成管理层 HTML 月报预览。
6. 系统能把月报发给指定收件人。

只有这六项都通过，才算邮箱接入成功。
