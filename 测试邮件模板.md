# 测试邮件模板

## 用途

下一版开发前，需要会计先发一封测试邮件，用来确认系统能否读取附件字段。

## 收件人

暂定：

`subsidy-ledger@synbiome.cn`

如果专用邮箱还没有创建，可以先发给负责人收集样表。

## 邮件标题

杭州微新：

`研发费用月度明细-2026-06-杭州微新`

深圳微智：

`研发费用月度明细-2026-06-深圳微智`

## 邮件正文

```
各位好，

附件为 2026 年 6 月研发费用明细账测试样表。

本邮件用于研发补贴台账系统测试：
1. 测试系统是否能读取附件；
2. 测试系统是否能识别研发费用；
3. 测试系统是否能区分杭州微新和深圳微智；
4. 测试系统是否能把不确定记录交给负责人审核。

请以附件数据为准。

谢谢。
```

## 附件要求

附件使用 Excel 或 CSV。

文件名建议：

- `研发费用明细-2026-06-杭州微新.xlsx`
- `研发费用明细-2026-06-深圳微智.xlsx`
- `研发费用明细-2026-06-杭州微新.csv`
- `研发费用明细-2026-06-深圳微智.csv`

## 测试通过标准

收到测试邮件后，下一版系统应能做到：

1. 读取邮件标题中的月份和主体。
2. 下载附件。
3. 识别研发费用科目。
4. 忽略明显非研发费用。
5. 把人员费用放到待审核。
6. 生成管理层 HTML 简报。
