# 邮件闭环完成度审计

## 审计结论

当前已经完成的是：

**邮件驱动闭环的下一版设计确认材料。**

当前还没有完成的是：

**真实自动收邮件、解析附件、更新云端台账、自动发送 CEO/CFO 邮件。**

这两件事必须区分清楚，不能把设计完成当成系统已经自动运行。

## 对照目标

目标链路：

1. 会计发邮件和附件。
2. 系统读取收件箱。
3. 系统解析附件。
4. 系统更新报表和台账。
5. 系统生成 HTML 结论。
6. 系统自动发给 CEO/CFO。

## 当前证据

### 已完成：设计边界

已有文件：

- `邮件驱动闭环设计说明.md`
- `邮件闭环实施计划.md`
- `邮件闭环设计确认单.md`
- `邮件闭环设计确认状态.md`
- `会计研发费用明细样表.csv`
- `会计样表填写说明.md`
- `测试邮件模板.md`
- `docs/superpowers/specs/2026-06-20-mail-driven-ledger-design.md`
- `docs/superpowers/plans/2026-06-20-mail-driven-ledger.md`

这些文件已经说明：

- 邮件和附件是输入源。
- 第一版只支持 Excel 和 CSV。
- 后台负责解析、匹配、审核和更新台账。
- CEO/CFO 只看 HTML 简报。
- 密码不能写进网页文件或 GitHub。
- 第一版不做完整会计系统直连。

### 已完成：管理层简报边界

现有网页已经保留管理层简报按钮。

管理层简报只显示：

- 预警等级。
- 研发投入达标率。
- 研发投入缺口。
- 补贴到账率。
- 三条行动建议。

明细、解析、费用口径和分摊都留在后台。

### 未完成：业务确认

还需要确认：

- 是否创建专用邮箱 `subsidy-ledger@synbiome.cn`。
- 是否能拿到专用邮箱授权密码。
- 月报最终发给哪些人。
- 是否固定每月 6 日上午 9 点发送。
- 会计是否能导出符合字段要求的样表。
- 是否确认张英审核不确定记录。

这些事项记录在：

`邮件闭环设计确认单.md`

### 未完成：真实自动化开发

目前还没有实现：

- 收件箱读取。
- 附件下载。
- Excel/CSV 后台解析。
- 自动生成导入批次。
- 待审核记录后台流转。
- 审核后写入云端台账。
- 定时生成 HTML 月报。
- 自动发邮件。

这些属于下一阶段开发工作。

## 不能声称已经完成的内容

目前不能说：

- 系统已经能自动收邮件。
- 系统已经能自动解析会计附件。
- 系统已经能自动更新云端台账。
- 系统已经能自动发送 CEO/CFO 月报。

可以说：

- 下一版边界已经定义清楚。
- 管理层简报保持简化。
- 开发计划和验收标准已经准备好。
- 会计样表和测试邮件模板已经准备好。
- 进入开发前需要先完成业务确认。

## 下一步进入开发的条件

满足以下三项后，可以开始开发邮件闭环：

1. 专用邮箱和授权密码准备好。
2. 会计提供一份真实导出的 Excel 或 CSV 样表。
3. 业务负责人确认 `邮件闭环设计确认单.md`。

## 推荐下一步动作

让业务负责人直接确认以下默认方案：

1. 使用 `subsidy-ledger@synbiome.cn` 作为专用邮箱。
2. 每月 6 日上午 9 点发送管理层简报。
3. 第一版只支持 Excel 和 CSV。
4. 张英审核不确定记录。
5. CEO/CFO 只看简报，不看明细。
