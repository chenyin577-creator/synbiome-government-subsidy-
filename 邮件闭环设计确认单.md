## 4. 数据进入方式

推荐优先级：

1. 会计系统 API。
2. 会计系统定时邮件或报表订阅。
3. 会计系统自动导出到云盘或对象存储。
4. 测试阶段人工发送邮件。

如果使用邮件，建议固定标题：

`研发费用月度明细-月份-主体`

示例：

- `研发费用月度明细-2026-06-杭州微新`
- `研发费用月度明细-2026-06-深圳微智`

确认项：

- [ ] 优先评估会计系统 API
- [ ] 优先评估定时邮件或报表订阅
- [ ] 优先评估自动导出到云盘或对象存储
- [ ] 暂时用人工测试邮件过渡

建议：

不要把人工发送邮件作为长期动作。如果必须先用测试邮件，应记录负责人和退出条件。

- Excel：`.xlsx`
- CSV：`.csv`

暂时不支持：

- PDF
- 图片
- Word
- 凭证压缩包

确认项：

- [ ] 第一版只支持 Excel 和 CSV
- [ ] 还需要支持 PDF
- [ ] 还需要支持其他格式：________________

建议：

第一版只支持 Excel 和 CSV。PDF、图片、凭证包会明显增加复杂度。

## 6. 附件字段

推荐会计导出的明细账至少包含：

- 日期
- 法人主体
- 会计科目
- 摘要
- 金额
- 凭证号
- 部门或项目

确认项：

- [ ] 会计可以导出上述字段
- [ ] 字段名称不完全一致，但含义一致
- [ ] 当前账套无法导出这些字段，需要先看样表

建议：

下一步让会计导出一份测试样表，用这份样表定最终字段映射。

## 7. 费用自动判断规则

推荐第一版规则：

- 会计科目含“研发费用”的，先进入候选研发费用。
- 明确写了项目编号或项目名称的，自动归集。
- 人员工资、薪酬、社保、公积金等人员费用，进入待审核。
- 科目不是研发费用但摘要疑似研发的，进入待审核。
- 完全无关费用，忽略但保留导入记录。

确认项：

- [ ] 按上述规则执行
- [ ] 只有系统不确定的费用才进入负责人审核
- [ ] 只自动识别研发费用科目，其他全部忽略

建议：

采用第一种。它兼顾效率和风险，人员费用等异常项仍由负责人审核；系统能判断的记录不再让人重复确认。

## 10. 第一版不做什么

推荐明确排除：

- 不做完整会计系统深度直连，但必须先查接口或自动导出能力。
- 不自动读取所有凭证和发票。
- 不处理 PDF 和图片票据。
- 不做多级审批流。
- 不做手机小程序。
- 不把前端确认作为每月固定流程。
- 不把人工下载、上传、复制、粘贴作为长期流程。

## 最终确认

如果以上默认推荐都可以，下一版就按以下结论开发：

1. 使用专用邮箱收发月报。
2. 优先通过会计系统接口、定时邮件或自动导出取得 Excel 或 CSV 明细账。
3. 系统自动识别研发费用。
4. 不确定记录由张英审核。
5. 每月 6 日上午 9 点发送管理层 HTML 简报。
6. 管理层只看预警和关键数字。
7. 前端只展示结果、预警和异常项。
8. 人工发送邮件只作为测试或临时兜底。

待确认事项：

- 专用邮箱是否已经创建。
- 专用邮箱是否已经生成授权密码。
- 会计能否提供一份真实导出的测试样表。
- 会计系统接口、定时导出或自动投递能力是否已经确认。
